Konsep utama

Object-Oriented Programming (OOP) adalah paradigma pemrograman yang mengorganisasi kode menjadi unit-unit bernama objek. Setiap objek memiliki atribut (data) dan metode (perilaku). OOP membuat kode lebih mudah dipahami, dirawat, dan dikembangkan ulang — prinsip yang digunakan hampir semua aplikasi modern.

Contoh kode — mendefinisikan kelas
mahasiswa.py 
# Mendefinisikan kelas Mahasiswa
class Mahasiswa:
    # Constructor — dipanggil saat objek dibuat
    def __init__(self, nama, nim):
        self.nama  = nama       # atribut publik
        self.__nim = nim        # atribut privat (enkapsulasi)

    def perkenalan(self):
        return f"Halo, saya {self.nama} — NIM {self.__nim}"

# Membuat objek (instance) dari kelas
mhs1 = Mahasiswa("Arini", "2024001")
print(mhs1.perkenalan())
# Output: Halo, saya Arini — NIM 2024001
💡 Tahukah kamu?
Instagram, YouTube, dan Tokopedia dibangun dengan prinsip OOP. Setiap "postingan", "video", atau "produk" adalah objek yang dibuat dari sebuah kelas. Bayangkan kelas Produk di Tokopedia — ia memiliki atribut harga, stok, nama, dan metode seperti tambahKeKeranjang().

Pewarisan (Inheritance) & Polimorfisme
⏱ Estimasi: 20 menit
🎯 Tujuan: Menerapkan inheritance dan polymorphism
📖 Penjelasan konsep

Inheritance memungkinkan kelas baru (child class) mewarisi atribut dan metode dari kelas yang sudah ada (parent class). Ini menghindari pengulangan kode dan mendukung prinsip DRY (Don't Repeat Yourself). Polymorphism memungkinkan metode yang sama berperilaku berbeda tergantung kelas objek yang memanggilnya.

hewan.py 
# Parent class (superclass)
class Hewan:
    def __init__(self, nama):
        self.nama = nama

    def suara(self):
        return "..."

# Child class — mewarisi Hewan (inheritance)
class Kucing(Hewan):
    def suara(self):        # override metode
        return "Meow!"

class Anjing(Hewan):
    def suara(self):
        return "Guk guk!"

# Polymorphism dalam aksi
hewan_list = [Kucing("Milo"), Anjing("Rex")]
for h in hewan_list:
    print(f"{h.nama}: {h.suara()}")
# Output:
# Milo: Meow!
# Rex: Guk guk!

Enkapsulasi & Abstraksi
⏱ Estimasi: 20 menit
🎯 Tujuan: Mengimplementasi encapsulation dan abstraction
📖 Penjelasan konsep

Encapsulation menyembunyikan detail internal objek — atribut privat hanya bisa diakses lewat metode khusus (getter/setter). Abstraction menyembunyikan kompleksitas dan hanya menampilkan antarmuka yang diperlukan. Kelas abstrak memaksa setiap child class untuk mengimplementasi metode yang diwajibkan.

pembayaran.py 
from abc import ABC, abstractmethod

# Abstraksi: kelas abstrak tidak bisa diinstansiasi langsung
class Pembayaran(ABC):
    def __init__(self, jumlah):
        self.__jumlah = jumlah      # enkapsulasi: atribut privat

    @property
    def jumlah(self):              # getter
        return self.__jumlah

    @abstractmethod
    def proses(self):              # wajib diimplementasi child
        pass

class QRIS(Pembayaran):
    def proses(self):
        return f"Bayar Rp{self.jumlah:,} via QRIS ✓"

class Transfer(Pembayaran):
    def proses(self):
        return f"Transfer Rp{self.jumlah:,} berhasil ✓"

trx = QRIS(150000)
print(trx.proses())
# Output: Bayar Rp150,000 via QRIS ✓
📊 Ringkasan 4 pilar OOP
🔒 Encapsulation
Menyembunyikan data internal. Akses hanya lewat getter/setter. Mencegah manipulasi data sembarangan dari luar kelas.
🌿 Inheritance
Child class mewarisi parent class. Kode tidak perlu ditulis ulang. Mendukung prinsip DRY dan hierarki kelas.
🔀 Polymorphism
Satu nama metode, banyak perilaku berbeda. Bergantung pada kelas objek yang memanggil metode tersebut.
🧩 Abstraction
Sembunyikan kompleksitas, tampilkan antarmuka. Kelas abstrak memaksa implementasi di setiap child class.
💡 Tahukah kamu?
Semua metode pembayaran digital di Indonesia — GoPay, OVO, DANA, ShopeePay — menggunakan pola abstraksi yang serupa. Ada satu antarmuka proses_pembayaran() yang diimplementasi berbeda-beda oleh tiap layanan, persis seperti contoh kode di atas!
