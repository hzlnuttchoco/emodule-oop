mahasiswa.py 
# Mendefinisikan kelas Mahasiswa
class Mahasiswa:
    def __init__(self, nama, nim, ipk):
        self.nama  = nama
        self.nim   = nim
        self.__ipk = ipk   # privat

    def get_ipk(self):
        return self.__ipk

    def perkenalan(self):
        return f"Halo! Saya {self.nama}, NIM {self.nim}, IPK {self.__ipk}"

mhs = Mahasiswa("Arini", "2024001", 3.87)
print(mhs.perkenalan())
print("IPK:", mhs.get_ipk())

Instagram & Tokopedia dibangun dengan OOP. Kelas Produk di Tokopedia punya atribut harga, stok, nama — dan metode seperti tambahKeKeranjang(). Setiap produk yang kamu lihat adalah objek dari kelas tersebut!

🎯 Menerapkan inheritance & polymorphism
📖 Penjelasan

Inheritance memungkinkan child class mewarisi atribut dan metode dari parent class, menghindari duplikasi kode. Polymorphism memungkinkan metode yang sama berperilaku berbeda tergantung kelas objeknya.

hewan.py 
class Hewan:
    def __init__(self, nama):
        self.nama = nama

    def suara(self):
        return "..."

    def info(self):
        return f"{self.nama} bersuara: {self.suara()}"

class Kucing(Hewan):      # inheritance
    def suara(self):        # override (polymorphism)
        return "Meow! 🐱"

class Anjing(Hewan):
    def suara(self):
        return "Guk guk! 🐶"

class Burung(Hewan):
    def suara(self):
        return "Cuit cuit! 🐦"

hewan_list = [Kucing("Milo"), Anjing("Rex"), Burung("Tweety")]
for h in hewan_list:
    print(h.info())

Enkapsulasi & Abstraksi
⏱ 20 menit
🎯 Implementasi encapsulation & abstraction
📖 Penjelasan

Encapsulation menyembunyikan data internal lewat atribut privat, diakses via getter/setter. Abstraction menyembunyikan kompleksitas — kelas abstrak memaksa child class mengimplementasi metode yang diwajibkan.

pembayaran.py 
class Pembayaran:
    def __init__(self, jumlah):
        self.__jumlah = jumlah  # privat

    def get_jumlah(self):   # getter
        return self.__jumlah

    def set_jumlah(self, val):  # setter dgn validasi
        if val > 0:
            self.__jumlah = val
        else:
            print("Jumlah harus positif!")

    def proses(self):          # akan di-override
        pass

class QRIS(Pembayaran):
    def proses(self):
        return f"✅ Bayar Rp{self.get_jumlah():,} via QRIS"

class Transfer(Pembayaran):
    def proses(self):
        return f"✅ Transfer Rp{self.get_jumlah():,} berhasil"

trx1 = QRIS(75000)
trx2 = Transfer(250000)
print(trx1.proses())
print(trx2.proses())
trx1.set_jumlah(-100)  # test validasi
💡 Tahukah kamu?
GoPay, OVO, DANA, ShopeePay — semuanya menggunakan pola yang sama! Ada satu antarmuka proses() yang diimplementasi berbeda-beda, persis seperti contoh di atas. Enkapsulasi juga menjaga saldo Anda tidak bisa diubah sembarangan dari luar!
